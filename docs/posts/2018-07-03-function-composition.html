Taking a page out of your high school algebra book

You may remember function composition from high school math:

g(f(x))‚ÄÑ=‚ÄÑ(g‚àòf)(x)

On the left hand side of the equation, we‚Äôre applying the function f to x (or in programming parlance, calling f with x as argument) and then applying g to the result. On the right hand side we are introducing the composition operator (little circle) and saying that the composed function g‚ÄÖ‚àò‚ÄÖf applied to x yields the same result.

It may not be obvious how that concept could be useful, but consider a case where the nesting of parentheses (or function calls, however you want to think of it) gets a bit out of hand:

j(i(h(g(f(x)))))‚ÄÑ=‚ÄÑ(j‚àòi‚àòh‚àòg‚àòf)(x)

In this case, composition allows us to think about a single composed function (j‚ÄÖ‚àò‚ÄÖi‚ÄÖ‚àò‚ÄÖh‚ÄÖ‚àò‚ÄÖg‚ÄÖ‚àò‚ÄÖf) that will be applied to an x. Contrast that with the left side where we have to first call f(x), and then pass the result to g, then pass that result to h and so on.

Real world application

Suppose you want to write a function that converts a roof pitch1 to degrees. E.g., for the input string "5:12", we want to return the number 22.6. A first attempt might go like so:

const pitchToDegrees = (str) =>
  Math.round(
    ((Math.atan(+str.split(/[:/]/)[0] / +str.split(/[:/]/)[1]) * 180) /
      Math.PI) *
      10
  ) / 10;

Which works alright, but it‚Äôs not so easy to follow how that initial string flows through each step of the function to produce the final result. You could try to make it more explicit by assigning each step to a variable:

const pitchToDegrees = (str) => {
  const splitStr = str.split(/[:/]/);
  const splitNums = splitStr.map(Number);
  const radians = Math.atan(splitNums[0] / splitNums[1]);
  const degrees = (radians * 180) / Math.PI;
  const rounded = Math.round(degrees * 10) / 10;
  return rounded;
};

But that gets pretty verbose, and since you‚Äôre just passing each result into the next expression, it might be more natural to read this as a composed function:

const pitchToDegree = compose(
  roundTo(1),
  radiansToDegrees,
  ([rise, run]) => Math.atan(rise / run),
  map(parseFloat),
  split(/[:/]/)
);

I think this strikes a balance between terseness and easy readability (once you get used to reading composed functions from right-to-left, bottom-to-top). It‚Äôs also been pointed out that this style of programming lends to more reliable, safer code, as it is harder to hack (in the sense of throwing things together in an undisciplined or naive manner). Every returned value has to necessarily flow into the next function; you really have to go out of your way to do any side effect-y things.2

It does require a little bit of setup beforehand with some helper functions:

const map = (fn) => (arr) => arr.map(fn);
const split = (regEx) => (str) => str.split(regEx);
const roundTo = (digits) => (num) =>
  Math.round(num * 10 ** digits) / 10 ** digits;
const radiansToDegrees = (rads) => (rads * 180) / Math.PI;

That might seem like a lot of work defining all these small, super simple functions, but that‚Äôs kind of the name of the game with functional programming, and the idea is that it pays off because all these little functions are easy to test and are usually very re-usable3. If you don‚Äôt feel like rolling your own tiny functions by hand, it‚Äôs easy enough to pull them in from a handy library like ramda or lodash/fp, which will have most of what you would need and more. We haven‚Äôt covered the one function that makes it possible to glue all the little functions together, a simple definition for which might look like:

const compose =
  (...fns) =>
  (x) =>
    fns.reduceRight((acc, fn) => fn(acc), x);

This definition can look pretty gnarly if you‚Äôre not used to looking at the array reduce method and the function that it takes as an argument, or the rest parameters syntax in JavaScript‚Äîbut I wouldn‚Äôt worry if it looks alien. If it helps, you can think of it as the more generalized version of what we‚Äôre doing here:

const compose2 = (g, f) => x => g(f(x))
const compose3 = (h, g, f) => x => h(g(f(x)))
const compose4 = (i, h, g, f) => x => i(h(g(f(x))))
...

Or you can just start using it and worry about the details behind how it‚Äôs implemented later, which tends to be how I learn. If you‚Äôre interested in learning more, this material is covered in the first five chapters of Professor Frisby Mostly Adequate Guide, or if you want to fast forward to trying out some composition exercises I have a repo of some here.

const trace = (msg) => (x) => {
  console.log(msg, x);
  return x;
};

Just drop it in in the middle of a composed function and you can have a peek at what‚Äôs being passed through at that point.
