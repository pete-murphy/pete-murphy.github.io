What is Functional Programming?

Before assuming that you want to learn FP, it may help to get an inkling as to what it is:

FP is just programming with functions. Functions are:

  1. Total: They return an output for every input.
  2. Deterministic: They return the same output for the same input.
  3. Pure: Their only effect is computing the output.

The rest is just composition you can learn over time.

Learning the Lambda Calculus

All functional programming languages have at their root the lambda calculus invented (or discovered?) by Alonzo Church in the 1930s. This is a subject worthy of study in its own right, but starting to think in terms of the abstractions involved in combinatory logic can be a very powerful exercise. I think the best introduction I‚Äôve found was in the first chapter of Haskell Book, but since that is not freely available, I‚Äôve tried to find some alternative resources.
